Obsidian is one of the most powerful local-first markdown note-taking apps available today. However, because it starts as a completely blank slate, it can feel incredibly overwhelming for beginners. Most people spend dozens of hours tweaking settings, themes, and plugins rather than actually writing notes.

In this guide, we are going to simplify the entire setup process. We will look at organizing directories, installing must-have plugins, and utilizing styles to transform Obsidian into a premium, focused workspace. If you want to skip the building process, our pre-configured Obsidian Starter Vault configuration file will be available here when the channel launches.

1. The Foundation: Folder Organization

A clutter-free sidebar is crucial to maintaining focus. We recommend a strict 4-folder structure to keep files tidy:

  • 00 Inbox: The landing pad for all new, unprocessed notes, web clips, and raw thoughts.
  • 10 Notes: The main repository where permanent, organized knowledge is stored.
  • 20 Templates: Storage for pre-written structures like meeting logs, weekly plans, and note metadata.
  • 90 System: The folder holding media attachments, custom CSS stylesheets, and vault assets.

"Keep your folder structure shallow. Lean on links and tags rather than nested subfolders to construct your web of knowledge."

2. Installing Essential Plugins

To expand Obsidian's functionality without slowing it down, we rely on three core community plugins:

  1. Dataview: Allows you to treat your notes like a database, writing queries to index lists automatically.
  2. Templater: Generates dynamic variables (like dates or titles) when creating new notes.
  3. Style Settings: Unlocks customization adjustments for custom themes directly in the settings UI.

Setting Up Dataview Indexes

For example, to list all notes containing the tag #project sorted by date modified, write the following query blocks: 

```dataview
TABLE file.folder AS "Folder", file.mtime AS "Last Modified"
FROM #project
SORT file.mtime DESC
```
